# 配置指南

本指南提供不同使用场景下的配置建议，帮助您根据实际需求选择最佳参数。

---

## 📋 快速开始

### 使用交互式配置工具（推荐）

```bash
python3 setup_config.py
```

工具会引导您完成以下步骤：
1. 选择使用场景
2. 配置浏览器显示模式
3. 选择网络环境
4. 设置懒加载策略
5. 自定义参数（可选）

### 手动配置

编辑 `playwright_crawler/config.py` 文件，修改相应参数。

## 🎯 使用场景推荐

### 场景 1: 快速测试

**适用情况：**
- 首次运行验证功能
- 测试少量数据（1-2页）
- 调试和开发阶段

**推荐配置：**
```python
HEADLESS = False                    # 显示浏览器窗口，便于观察
PAGE_LOAD_WAIT = (2000, 3000)       # 2-3秒，快速加载
SCROLL_COUNT = (3, 5)              # 3-5次滚动，够用即可
BETWEEN_PAGES = (2000, 3000)       # 2-3秒，快速切换
BETWEEN_TASKS = (2000, 3000)       # 2-3秒，快速切换
```

**预期效果：**
- 每页约 5-8 秒
- 10 页约需 1-2 分钟
- 适合快速验证和调试

---

### 场景 2: 小规模爬取

**适用情况：**
- 个人使用或小项目
- 爬取 10-20 个分类
- 需要平衡速度和稳定性

**推荐配置：**
```python
HEADLESS = False                    # 可选：显示窗口观察进度
PAGE_LOAD_WAIT = (3000, 5000)       # 3-5秒，确保加载完成
SCROLL_COUNT = (5, 8)              # 5-8次滚动，标准配置
BETWEEN_PAGES = (5000, 10000)      # 5-10秒，避免频繁请求
BETWEEN_TASKS = (5000, 10000)      # 5-10秒，避免触发反爬
```

**预期效果：**
- 每页约 10-15 秒
- 50 页约需 10-15 分钟
- 适合日常使用

---

### 场景 3: 大规模爬取

**适用情况：**
- 爬取所有分类（88个）
- 批量数据采集
- 需要长期稳定运行

**推荐配置：**
```python
HEADLESS = True                     # 无头模式，节省资源
PAGE_LOAD_WAIT = (5000, 8000)       # 5-8秒，充分等待
SCROLL_COUNT = (5, 8)              # 5-8次滚动，确保完整
BETWEEN_PAGES = (10000, 15000)     # 10-15秒，降低请求频率
BETWEEN_TASKS = (10000, 15000)     # 10-15秒，避免被封
```

**预期效果：**
- 每页约 15-20 秒
- 100 页约需 30-40 分钟
- 88个分类（约200页）约需 1-1.5 小时

---

### 场景 4: 服务器运行

**适用情况：**
- 在无头服务器上运行
- 无人值守长期运行
- 需要最小化资源占用

**推荐配置：**
```python
HEADLESS = True                     # 必须启用无头模式
PAGE_LOAD_WAIT = (8000, 12000)     # 8-12秒，保守策略
SCROLL_COUNT = (5, 8)              # 5-8次滚动
BETWEEN_PAGES = (15000, 20000)     # 15-20秒，极度保守
BETWEEN_TASKS = (15000, 20000)     # 15-20秒，确保稳定
```

**预期效果：**
- 每页约 20-25 秒
- 100 页约需 40-50 分钟
- 最稳定，但速度最慢

## 🔧 参数详解

### HEADLESS（无头模式）

| 值 | 说明 | 适用场景 |
|----|------|---------|
| `False` | 显示浏览器窗口 | 测试、调试、观察进度 |
| `True` | 不显示窗口 | 服务器、后台运行、大规模爬取 |

**建议：**
- 首次使用：`False`（观察运行状态）
- 正式运行：`True`（节省资源）

---

### PAGE_LOAD_WAIT（页面加载等待）

页面加载完成后的随机等待时间（毫秒）。

| 网络环境 | 推荐值 | 说明 |
|---------|--------|------|
| 高速（国内宽带/企业） | 2000-5000 | 快速加载 |
| 普通（家庭宽带） | 3000-7000 | 适中等待 |
| 较慢（海外/不稳定） | 5000-12000 | 充分等待 |

**作用：**
- 确保页面完全加载
- 模拟真实用户行为
- 避免反爬检测

---

### SCROLL_COUNT（滚动次数）

触发懒加载的滚动次数（随机范围）。

| 策略 | 推荐值 | 优缺点 |
|------|--------|--------|
| 保守 | (3, 5) | 快速，但可能遗漏部分商品 |
| 平衡 | (5, 8) | 推荐设置，兼顾速度和完整性 |
| 激进 | (8, 12) | 确保加载所有商品，但较慢 |

**建议：**
- 测试阶段：`(3, 5)`
- 正式爬取：`(5, 8)`
- 网络较慢：`(8, 12)`

---

### BETWEEN_PAGES（页面间延时）

同一分类中不同页面之间的延时（毫秒）。

| 网络环境 | 推荐值 | 说明 |
|---------|--------|------|
| 高速 | 2000-10000 | 快速切换 |
| 普通 | 5000-15000 | 适中延时 |
| 较慢 | 10000-20000 | 充分延时 |

**作用：**
- 避免短时间内多次请求
- 降低被封风险
- 模拟真实浏览间隔

---

### BETWEEN_TASKS（任务间延时）

不同分类之间的延时（毫秒）。

| 网络环境 | 推荐值 | 说明 |
|---------|--------|------|
| 高速 | 2000-10000 | 快速切换 |
| 普通 | 5000-15000 | 适中延时 |
| 较慢 | 10000-20000 | 充分延时 |

**作用：**
- 避免短时间内访问不同分类
- 进一步降低被封风险

## 🛠️ 高级配置

### SLOW_MO（慢动作模式）

```python
SLOW_MO = 0  # 毫秒
```

- `0`：正常速度
- `100`：每个操作延迟 100ms，便于观察
- `500`：每个操作延迟 500ms，适合调试

**建议：**
- 调试时设置 `500`，观察每个步骤
- 正式运行保持 `0`

---

### TIMEOUT（超时时间）

```python
TIMEOUT = 60000  # 60秒
```

页面操作的超时时间（毫秒）。

**建议：**
- 正常网络：`60000`（60秒）
- 较慢网络：`120000`（120秒）
- 海外访问：`180000`（180秒）

## ⚠️ 注意事项

### 1. 网络不稳定

如果遇到频繁超时或失败：
- 增加 `PAGE_LOAD_WAIT`
- 增加 `BETWEEN_PAGES`
- 增加 `TIMEOUT`

### 2. 被反爬虫检测

如果发现爬取失败率变高：
- 增加 `BETWEEN_PAGES`
- 增加 `BETWEEN_TASKS`
- 降低 `SCROLL_COUNT`
- 启用无头模式（如果未启用）

### 3. 提取商品数不足

如果每页提取的商品数明显少于预期：
- 增加 `SCROLL_COUNT`
- 增加 `PAGE_LOAD_WAIT`
- 检查网络是否太慢

### 4. 内存占用过高

如果运行时内存占用过大：
- 启用无头模式 `HEADLESS = True`
- 减少同时运行的爬虫实例
- 关闭其他占用内存的程序

## 🔍 故障排查

### 问题：频繁超时

**可能原因：** 网络太慢，页面加载超时

**解决方案：**
```python
# 增加页面加载等待
PAGE_LOAD_WAIT = (8000, 12000)

# 增加超时时间
TIMEOUT = 120000
```

---

### 问题：被反爬虫限制

**可能原因：** 请求频率过高

**解决方案：**
```python
# 增加所有延时
PAGE_LOAD_WAIT = (5000, 8000)
BETWEEN_PAGES = (15000, 20000)
BETWEEN_TASKS = (15000, 20000)

# 启用无头模式
HEADLESS = True
```

---

### 问题：商品数太少

**可能原因：** 懒加载触发不足

**解决方案：**
```python
# 增加滚动次数
SCROLL_COUNT = (8, 12)

# 增加页面等待时间
PAGE_LOAD_WAIT = (5000, 8000)
```
